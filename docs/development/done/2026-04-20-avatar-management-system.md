# Avatar Management System Implementation Plan

**Goal:** Provide users with full control over their profile avatars, allowing for custom uploads to R2 and selection from a curated library of Divemap-owned avatars.

**Architecture:**
1.  **Storage:** Use Cloudflare R2 for all avatars.
    - System Avatars: `library/avatars/`
    - User Avatars: `avatars/user_{id}/`
2.  **Library:** A static set of 50 humorous scuba-themed animal avatars generated by `nanobanana`.
3.  **Ownership:** Custom avatars are private to the user; the API ensures only the owner can use their uploaded paths.
4.  **Google Auth:** Users get an initial avatar and its URL is backed up to `google_avatar_url` to allow for easy "reset to default" functionality.
5.  **Robust Fallbacks:** The `Avatar` component handles loading errors (e.g., Google 429s) by falling back to colored initials.
6.  **Optimization:** All library avatars are optimized to 512x512 WebP format (~30KB per icon) to minimize bandwidth.

**Tech Stack:**
- Backend: FastAPI, SQLAlchemy, Pydantic, `boto3` (R2), `nh3` (Sanitization)
- Frontend: React, `react-hook-form`, `react-query`, Lucide icons
- Image Generation: `nanobanana`
